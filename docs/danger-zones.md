# Danger Zones

## Overview

Displays crime statistics and official zone overlays for the Ile-de-France region on the map. Three independent layers are available:

- **SSMSI crime data** — choropleth or heatmap per commune/arrondissement, with 16 selectable indicators
- **QPV** — Quartiers Prioritaires de la politique de la Ville (priority neighborhoods)
- **QRR** — Quartiers de Reconquête Républicaine (republican reconquest zones)

## Data Sources

| Source | Provider | Scope |
|--------|----------|-------|
| Crime statistics | SSMSI via [data.gouv.fr](https://data.gouv.fr) | 1,285 communes + arrondissements |
| Commune boundaries | `geo.api.gouv.fr` | 8 IDF departments |
| Paris arrondissements | [Paris Open Data](https://opendata.paris.fr) | 20 arrondissements |
| QPV polygons | data.gouv.fr | 298 zones in IDF |
| QRR polygons | data.gouv.fr | 16 zones in IDF |

Paris is split into 20 arrondissements (codes 75101–75120) instead of being displayed as a single commune.

## Indicators

The SSMSI dataset provides 16 indicators per commune:

| ID | EN | FR |
|----|----|----|
| `all` | All crimes | Tous les faits |
| `burglary` | Burglary | Cambriolages |
| `vandalism` | Vandalism | Destructions et dégradations |
| `fraud` | Fraud | Escroqueries et fraudes |
| `drug_trafficking` | Drug trafficking | Trafic de stupéfiants |
| `drug_use` | Drug use | Usage de stupéfiants |
| `physical_violence` | Physical violence | Violences physiques |
| `domestic_violence` | Domestic violence | Violences intrafamiliales |
| `sexual_violence` | Sexual violence | Violences sexuelles |
| `armed_robbery` | Armed robbery | Vols avec armes |
| `vehicle_accessory_theft` | Vehicle accessory theft | Vols d'accessoires sur véhicules |
| `theft_from_vehicles` | Theft from vehicles | Vols dans les véhicules |
| `vehicle_theft` | Vehicle theft | Vols de véhicule |
| `theft_no_violence` | Theft without violence | Vols sans violence |
| `violent_theft` | Violent theft | Vols violents sans arme |

Crime rates are computed per 1,000 inhabitants: `indicators[id] / population * 1000`.

## Data Pipeline

All data is embedded as static JSON files generated by a build script.

### Generation

```bash
npm run prepare:crime-data
```

This runs `scripts/prepare-crime-data.ts` which:

1. Downloads SSMSI crime CSV from data.gouv.fr (streaming parser)
2. Downloads commune GeoJSON boundaries from `geo.api.gouv.fr`
3. Downloads Paris arrondissements GeoJSON from Paris Open Data
4. Downloads QPV GeoJSON from data.gouv.fr (ZIP, WGS84 file)
5. Downloads QRR Shapefile from data.gouv.fr (ZIP, converted via `shpjs`)
6. Replaces Paris (75056) with 20 arrondissements in the commune dataset
7. Simplifies all geometries with `@turf/simplify`
8. Filters QPV/QRR to IDF departments only
9. Writes 3 output files

### Output Files

| File | Size | Content |
|------|------|---------|
| `data/idf-crime-data.json` | ~1.1 MB | 1,285 communes with per-indicator values |
| `data/qpv-idf.json` | ~0.2 MB | 298 QPV polygons |
| `data/qrr-idf.json` | ~0.02 MB | 16 QRR polygons |

## Render Modes

### Choropleth

Colors each commune/arrondissement polygon based on its crime rate for the selected indicator. Uses a green-yellow-red gradient (RdYlGn color scale).

- **Native**: `react-native-maps` `Polygon` components
- **Web**: `react-leaflet` `GeoJSON` with style function and hover tooltips

### Heatmap

Renders an interpolated heat layer using commune centroids weighted by crime rate.

- **Native**: `react-native-maps` `Circle` components with opacity proportional to weight
- **Web**: `leaflet.heat` plugin with gradient overlay

### QPV Overlay

Displays QPV zone boundaries with blue stroke and semi-transparent blue fill. Independent of the SSMSI toggle.

### QRR Overlay

Displays QRR zone boundaries with purple stroke and semi-transparent purple fill. Independent of the SSMSI toggle.

## Hook

`useDangerZones()` manages all danger zone state:

```typescript
const {
  isVisible,            // boolean — SSMSI layer visible
  renderMode,           // 'choropleth' | 'heatmap'
  selectedIndicator,    // string — current indicator ID
  showQPV,              // boolean — QPV overlay visible
  showQRR,              // boolean — QRR overlay visible
  communeData,          // CommuneRenderData[] — colors for selected indicator
  heatmapPoints,        // HeatmapPoint[] — centroids for selected indicator
  availableIndicators,  // IndicatorMeta[] — list of selectable indicators
  qpvData,              // QPVFeature[]
  qrrData,              // QRRFeature[]
  dataYear,             // number
  toggleVisibility,     // () => void
  toggleRenderMode,     // () => void
  setIndicator,         // (id: string) => void
  toggleQPV,            // () => void
  toggleQRR,            // () => void
} = useDangerZones();
```

`communeData` and `heatmapPoints` are recomputed via `useMemo` when `selectedIndicator` changes. QPV/QRR data is static.

## Configuration

`DEFAULT_DANGER_ZONE_CONFIG` in `types/dangerZone.ts`:

| Property | Value | Description |
|---|---|---|
| `maxIntensity` | 1 | Heatmap max intensity |
| `radiusMeters` | 4000 | Native circle radius |
| `opacity` | 0.6 | Choropleth fill opacity |
